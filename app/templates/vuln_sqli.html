2. SQLインジェクション

概要

ユーザーの入力値がSQL文に直接埋め込まれると、攻撃者が任意のSQLを実行できます。 認証バイパス、データ窃取、データ改ざん・削除などに悪用されます。

体験: ログインフォーム（SQLi用）

攻撃ペイロード例
  • 認証バイパス: ユーザー名 admin' --（パスワードは何でもOK）
  • 全ユーザー取得: ユーザー名 ' OR '1'='1
  • UNION SELECT: ユーザー名 ' UNION SELECT 1,username,password,email,role,credit_card,secret_note FROM users --

背後のSQLは SELECT * FROM users WHERE username = '入力値' AND password = '入力値' です。 シングルクォートでSQL文の構造を壊せないか考えてみましょう。

SQLでは -- 以降はコメントとして無視されます。 admin' -- と入力すると、パスワードチェックが無効化されます。

UNION SELECT を使えば、別のテーブルやカラムからもデータを取得できます。 まずカラム数を合わせる必要があります（このテーブルは7カラム）。
